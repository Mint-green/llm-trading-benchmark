# 当前技术方案相较上一版的差异与改进点

## 1. 总体变化

上一版系统的核心形态是：

```text
每 5min 构造一次上下文
给模型市场规则、完整 universe、候选榜、持仓、工具说明
模型 query 或 trade
系统执行并记录指标
```

当前版改为：

```text
每 5min 系统扫描
30min full decision
事件触发 focused decision
结构化 active plan / memory state
候选分桶
查询工具原生化
最终决策 JSON 化
执行与记忆由系统管理
```

核心改进目标是降低随机性、减少 prompt 噪声、增强决策连续性、提高可审计性。

------

# 2. 决策频率改进

## 上一版

```text
5min Decision Frequency
每个 5min 都可能让模型完整判断一次
```

问题：

```text
模型容易失忆重启
短期噪声变多
HOLD 与随机交易都可能增加
token 与延迟成本高
```

## 当前版

```text
5min scanner
30min full decision
event-driven focused decision
auto_hold
```

好处：

```text
5min 仍然监控市场
但不每 5min 重启完整思考
真正有事件才触发小决策
常规 30min 决策负责全局重评
```

------

# 3. Universe 暴露方式改进

## 上一版

首屏提供完整 universe ticker 列表。

问题：

```text
信息密度低
token 占用高
模型容易按熟悉 ticker 猜
A股/HK ticker 对模型帮助有限
```

## 当前版

首屏不展开完整 universe，只显示：

```text
分桶候选
持仓
watchlist
avoid
blocked/warning
market summary
```

完整 universe 仍然存在，但通过 `screen_universe` 工具探索。

好处：

```text
减少首屏噪声
保留全市场探索能力
模型可以表达筛选意图
```

------

# 4. Candidate Layer 改进

## 上一版

单一候选榜：

```text
rank|ticker|score|price|1h|1d|5d|rsi|trend|tradable
```

问题：

```text
模型容易只看前几名
前几名往往高涨幅/高 RSI
追高与 HOLD 摇摆
不同资产类型混排
```

## 当前版

分桶候选：

```text
held_positions
exit_watch
trend_leaders
pullback_continuation
oversold_reversal
low_vol_defensive
crypto_candidates
futures_macro
blocked_or_warning
```

好处：

```text
模型先理解“机会类型”
不是只看总分排名
crypto/futures 不与股票混排
危险标的明确提示
```

------

# 5. 工具设计改进

## 上一版

通过 prompt 描述伪工具：

```json
{"action":"query","queries":[{"tool":"query_stock","args":{...}}]}
```

问题：

```text
工具名/参数可能写错
模型多轮查询成本高
查询和决策边界不清
```

## 当前版

查询类工具使用原生 function/tool calling：

```text
screen_universe
query_asset
query_position
query_history
query_market_overview
query_fx
query_futures_contract
query_news 预留
```

交易和记忆更新不做工具，而在最终 JSON 中表达。

好处：

```text
工具参数 schema 可校验
模型只用工具读信息
交易和状态由系统执行
边界更清晰
```

------

# 6. 输出协议改进

## 上一版

```json
{
  "action": "trade",
  "trades": [
    {
      "ticker": "AAPL.US",
      "side": "buy",
      "allocation_pct": 0.05
    }
  ]
}
```

问题：

```text
SELL 的 allocation_pct 语义不清
是卖 NAV 比例，还是卖持仓比例？
模型样例中已经出现自我猜测
```

## 当前版

目标仓位制：

```json
{
  "portfolio_targets": [
    {
      "symbol": "2318.HK",
      "target_pct_nav": 0.03
    },
    {
      "symbol": "sh.600183",
      "target_pct_nav": 0.00
    }
  ]
}
```

好处：

```text
买卖统一成目标仓位
减仓/清仓无歧义
执行层负责单位换算
更适合多账户规模和多市场
```

------

# 7. 记忆系统改进

## 上一版

主要依靠 prompt 当前快照和最近失败交易提醒。

问题：

```text
模型没有连续计划
每 5min 近似重启
观察名单、禁买、持仓计划难以延续
```

## 当前版

结构化记忆：

```text
active_plans
plan_triggers
watchlist
avoid_list
daily_thesis
recent_activity
execution_feedback
session_summary
daily_summary
```

好处：

```text
模型知道之前为什么买
知道上次 focused decision 怎么处理
知道哪些标的在观察
知道哪些标的暂时避免
知道前一日总结
```

------

# 8. Focused Decision 新增

## 上一版

所有问题都进入完整决策。

## 当前版

持仓事件单独触发 focused decision：

```text
止损复查
止盈复查
trailing stop
bars_elapsed
市场关闭前复查
保证金 warning
```

好处：

```text
prompt 更短
只处理相关资产
更像真实盯盘
减少无关市场噪声
```

------

# 9. 状态触发器改进

## 上一版

止损/止盈主要靠 prompt 文本规则。

问题：

```text
自然语言无法稳定转代码
规则不可审计
模型可能每次理解不同
```

## 当前版

结构化 trigger：

```json
{
  "type": "price_move_pct",
  "direction": "down",
  "anchor": "last_review_price",
  "threshold_pct": 0.01
}
```

支持：

```text
price_move_pct
atr_move
pnl_pct
trailing_drawdown_pct
trailing_atr
bars_elapsed
regime_change
asset_status_change
margin_risk_change
```

好处：

```text
代码可计算
回测可复现
触发原因可归因
```

------

# 10. 数据库与审计改进

## 上一版

主要记录交易、指标、拒单、行为指标。

## 当前版

完整记录：

```text
prompt_snapshot
tool_schema_hash
tool_calls
raw_model_output
parsed_output
validation_result
execution_result
state_diff
plan_versions
memory_versions
summary_json
portfolio_snapshots
metrics_daily
```

好处：

```text
可以完整回放单模型路径
可以比较不同模型的状态演化
可以分析某笔收益来自候选、工具、记忆还是 focused decision
```

------

# 11. 每日总结改进

## 上一版

没有完整记忆总结机制，新闻后置。

## 当前版

新增：

```text
session_summary
daily_global_summary
previous_daily_summary 注入次日第一次 full decision
rolling_behavior_notes
```

总结使用统一 summarizer，而不是被测交易模型自己总结。

好处：

```text
次日不失忆
但不会把前一天所有 prompt 堆进去
减少模型间总结能力差异
```

------

# 12. 新闻模块处理差异

## 上一版

query_news 预留。

## 当前版

仍然预留，但不进入主榜默认流程。

原因：

```text
新闻泄露风险高
数据源覆盖不均
摘要可能隐含未来反应
噪音可能增加换手
贡献需要消融验证
```

后续作为 ablation：

```text
price-only
price + news digest
price + query_news
delayed_news
shuffled_news
```

------

# 13. 当前版的核心收益

```text
更公平：
- 工具 schema 固定
- 记忆由系统管理
- 总结模型统一
- prompt 构造可版本化

更可复现：
- 每次 prompt、tool、response、execution、state_diff 都落库
- trigger 可计算
- replay 可重建

更接近散户：
- 有行情首屏
- 有筛选器
- 有持仓计划
- 有观察/禁买
- 有盯盘事件
- 有每日复盘

更稳定：
- 不再每 5min 完整重启
- 减少无意义交易
- focused decision 降低噪声
- active plan 维持连续性
```

------

# 14. 最大代价

当前版比上一版复杂，主要代价是：

```text
StateManager 复杂
数据库表更多
prompt builder 更复杂
replay 要求更高
trigger schema 需要严格实现
```

但复杂度主要在系统侧，不压给模型。模型看到的是简短交易工作台，而不是庞大状态机。
